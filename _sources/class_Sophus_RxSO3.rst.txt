Returns the ith infinitesimal generators of ``R+ x SO(3)``.

The infinitesimal generators of :ref:`RxSO3 <doxid-class_sophus_1_1_rx_s_o3>` are:

.. ref-code-block:: cpp

	      |  0  0  0 |
	G_0 = |  0  0 -1 |
	      |  0  1  0 |
	
	      |  0  0  1 |
	G_1 = |  0  0  0 |
	      | -1  0  0 |
	
	      |  0 -1  0 |
	G_2 = |  1  0  0 |
	      |  0  0  0 |
	
	      |  1  0  0 |
	G_2 = |  0  1  0 |
	      |  0  0  1 |

Precondition: ``i`` must be 0, 1, 2 or 3.

.. _doxid-class_sophus_1_1_rx_s_o3_1ab79f419758d755bd08a447ffefff46ee:

.. ref-code-block:: cpp

	static :ref:`Transformation<doxid-class_sophus_1_1_rx_s_o3_base_1aee2958a758e4940a5b9b927a58daf8e0>` hat(:ref:`Tangent<doxid-class_sophus_1_1_rx_s_o3_base_1aca5a75b2238ad6d68ae9b7a3a7e82afe>` const& a)

hat-operator

It takes in the 4-vector representation ``a`` (= rotation vector plus logarithm of scale) and returns the corresponding matrix representation of Lie algebra element.

Formally, the :ref:`hat() <doxid-class_sophus_1_1_rx_s_o3_1ab79f419758d755bd08a447ffefff46ee>` -operator of :ref:`RxSO3 <doxid-class_sophus_1_1_rx_s_o3>` is defined as

``hat(.): R^4 -> R^{3x3}, hat(a) = sum_i a_i * G_i`` (for i=0,1,2,3)

with ``G_i`` being the ith infinitesimal generator of :ref:`RxSO3 <doxid-class_sophus_1_1_rx_s_o3>`.

The corresponding inverse is the :ref:`vee() <doxid-class_sophus_1_1_rx_s_o3_1a0505328a9c250b3f786febb157a05ca5>` -operator, see below.

.. index:: pair: function; lieBracket
.. _doxid-class_sophus_1_1_rx_s_o3_1a41c045464b987c9d6a56421dabb484eb:

.. ref-code-block:: cpp
	:class: doxyrest-title-code-block

	static :ref:`Tangent<doxid-class_sophus_1_1_rx_s_o3_base_1aca5a75b2238ad6d68ae9b7a3a7e82afe>` lieBracket(:ref:`Tangent<doxid-class_sophus_1_1_rx_s_o3_base_1aca5a75b2238ad6d68ae9b7a3a7e82afe>` const& a, :ref:`Tangent<doxid-class_sophus_1_1_rx_s_o3_base_1aca5a75b2238ad6d68ae9b7a3a7e82afe>` const& b)

Lie bracket.

It computes the Lie bracket of RxSO(3). To be more specific, it computes

``[omega_1, omega_2]_rxso3 := vee([hat(omega_1), hat(omega_2)])``

with ``[A,B] := AB-BA`` being the matrix commutator, ``hat(.)`` the :ref:`hat() <doxid-class_sophus_1_1_rx_s_o3_1ab79f419758d755bd08a447ffefff46ee>` -operator and ``vee(.)`` the :ref:`vee() <doxid-class_sophus_1_1_rx_s_o3_1a0505328a9c250b3f786febb157a05ca5>` -operator of :ref:`RxSO3 <doxid-class_sophus_1_1_rx_s_o3>`.

vee-operator

It takes the 3x3-matrix representation ``Omega`` and maps it to the corresponding vector representation of Lie algebra.

This is the inverse of the :ref:`hat() <doxid-class_sophus_1_1_rx_s_o3_1ab79f419758d755bd08a447ffefff46ee>` -operator, see above.

Precondition: ``Omega`` must have the following structure:

.. code-block:: cpp

	|  d -c  b |
	|  c  d -a |
	| -b  a  d |
